# Phase 1: PRD (Product Requirements Document)

<div class="grid grid-cols-2 gap-8 mt-8">

<div>

## What Is It?

The **business requirements** for your application

### Contains
- User stories
- Feature descriptions
- Technical constraints
- Success criteria
- Target audience

</div>

<div>

## Example: FIFA Navigator

```markdown
## User Story
As a FIFA fan visiting Atlanta,
I want to see real-time transit to the stadium
So I can arrive on time without driving

## Features
- Google Maps with traffic layer
- MARTA bus & train real-time data
- Match schedule
- Multi-language support (EN/ES/DE/KO)

## Constraints
- Must work in GitHub Codespaces
- MARTA Train API on port 18096 (blocked)
- Must be mobile-responsive
```

</div>

</div>
