# The 8 Frameworks

<div class="grid grid-cols-2 gap-4 mt-8">

<div class="p-4 bg-blue-500/10 rounded-lg">

### S.P.A.R.K
**Educational Setup**
- Situation, Purpose, Audience, Roadmap, Knowledge
- Use for: Project initialization, learning-focused tasks

</div>

<div class="p-4 bg-orange-500/10 rounded-lg">

### F.O.C.U.S
**Constraint Solving**
- Frame, Objective, Constraints, Users, Steps
- Use for: Environment-specific problems (Codespaces proxy!)

</div>

<div class="p-4 bg-purple-500/10 rounded-lg">

### C.R.A.F.T
**Technical Integration**
- Context, Role, Action, Format, Tone
- Use for: API integrations, library setup

</div>

<div class="p-4 bg-pink-500/10 rounded-lg">

### I.D.E.A
**Creative/UX Design**
- Issue, Details, Example, Action
- Use for: UI components, user interactions

</div>

<div class="p-4 bg-green-500/10 rounded-lg">

### S.T.A.G.E
**Step-by-Step Implementation**
- Scenario, Task, Approach, Guardrails, Execution
- Use for: Multi-phase processes, data seeding

</div>

<div class="p-4 bg-yellow-500/10 rounded-lg">

### M.A.P
**Metrics & Planning**
- Metrics, Actions, Proof
- Use for: Acceptance criteria, success definition

</div>

<div class="p-4 bg-teal-500/10 rounded-lg">

### P.R.O.M.P.T
**Testing Strategy**
- Problem, Roles, Objectives, Methods, Process, Timeframe
- Use for: Test plans, verification strategies

</div>

<div class="p-4 bg-red-500/10 rounded-lg">

### D.E.C.I.D.E
**Decision Making**
- Define, Explore, Consider, Identify, Decide, Evaluate
- Use for: Tech stack choices, architecture decisions

</div>

</div>
